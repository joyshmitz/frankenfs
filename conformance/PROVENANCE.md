# Fixture Provenance

> Records how each fixture was generated for reproducibility.

## Generation Method

All fixtures are hand-crafted JSON files that encode sparse binary data using the
`{"offset": N, "hex": "..."}` write format. The binary is reconstructed by
`load_sparse_fixture()` in ffs-harness.

**Reference source:** Linux kernel v6.19 header files and on-disk format documentation.
- ext4: `fs/ext4/ext4.h`, ext4.wiki.kernel.org
- btrfs: `fs/btrfs/ctree.h`, `fs/btrfs/btrfs_tree.h`, btrfs.readthedocs.io

## Golden Files

Golden files in `conformance/golden/` are generated from real filesystem images:

### ext4_*.json
- **Generated by:** `ffs-cli inspect --format json <image>`
- **Source images:** Created via `mkfs.ext4` with various options
- **mkfs version:** e2fsprogs 1.47.0
- **Command:** `mkfs.ext4 -L test -b 4096 <image> <size>`

### btrfs_*.json
- **Generated by:** `ffs-cli inspect --format json <image>`
- **Source images:** Created via `mkfs.btrfs` with various options
- **mkfs version:** btrfs-progs 6.6.3
- **Command:** `mkfs.btrfs -L test <image>`

## Checksum Verification

All fixtures have SHA-256 checksums recorded in `fixtures/checksums.sha256`.
Regenerate with: `sha256sum conformance/fixtures/*.json > conformance/fixtures/checksums.sha256`
